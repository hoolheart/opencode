# 深度调研代理群使用指南

## 概述

深度调研代理群（Deep Research Agent Group）是一套专门用于执行系统性、高质量调研任务的 AI 代理系统。通过 6 个专业化代理的分工协作，能够完成从需求澄清到最终报告交付的完整调研流程。

## 核心特点

- **严格的质量控制**: 多轮审核机制，确保事实准确性
- **完整的引用追溯**: 所有论断都必须有原始来源支持
- **创造性扩展能力**: 自动识别盲区并提出新的调研方向
- **标准化流程**: 8 阶段工作流程，确保调研完整性
- **PDF 支持**: 学术论文等文档自动处理与保存

## 代理组成

### 1. Research（主代理）

**类型**: Primary  
**温度**: 0.5  
**职责**:
- 接收并澄清用户的调研需求
- 设计问卷辅助用户明确细节
- 协调其他子代理完成调研任务
- 管理整个调研工作流程
- 输出最终的调研报告

**输入**: 用户的调研需求（领域、方向、目标）  
**输出**: request.md、standard.md、final_report.md

### 2. Inspect（查询设计代理）

**类型**: Subagent  
**温度**: 0.3  
**职责**:
- 根据需求设计全面的搜索关键词
- 识别重点查询网站和信息源
- 制定搜索策略，确保信息覆盖度

**输入**: request.md、revise.md（如有）  
**输出**: search_plan.md

### 3. Search（信息收集代理）

**类型**: Subagent  
**温度**: 0.3  
**职责**:
- 执行搜索计划，收集网络信息
- 将原始信息转写为结构化 Markdown
- 保存原始 PDF 文件到 raw/pdfs/
- 维护完整的来源引用

**输入**: search_plan.md  
**输出**: raw/*.md、raw/pdfs/*.pdf

### 4. Insight（报告编写代理）

**类型**: Subagent  
**温度**: 0.4  
**职责**:
- 分析原始调研材料
- 综合关键洞见，识别模式
- 编写结构化的调研报告
- 确保所有论断都有准确引用

**输入**: request.md、raw/*.md、revise.md（如有）  
**输出**: result.md

### 5. Creative（创造性思考代理）

**类型**: Subagent  
**温度**: 0.7（创造性设置）  
**职责**:
- 识别调研中的盲区和信息缺口
- 提出替代视角和角度
- 建议新的调研方向和方法
- 跨学科联想，寻找创新思路

**输入**: revise.md、result.md、request.md  
**输出**: creative_directions.md

### 6. Examine（质量审核代理）

**类型**: Subagent  
**温度**: 0.0（完全客观）  
**职责**:
- 严格审核调研报告
- 验证每个引用的准确性（引用不符→大修）
- 检查逻辑一致性和论证有效性
- 判定修订级别：大修/小修/通过

**输入**: standard.md、result.md、raw/*.md  
**输出**: examine_report.md、revise.md（如需修订）

## 工作流程

### 阶段 1: 需求澄清
1. **Research** 接收用户调研需求
2. 判断是否足够具体（能否生成 request.md 和 standard.md）
3. 如不够具体，设计问卷与用户确认细节

### 阶段 2: 规划
1. **Research** 创建 `request.md`（调研需求文档）
2. **Research** 创建 `standard.md`（审核标准文档）

### 阶段 3: 信息收集
1. **Research** → **Inspect**: 根据 request.md 设计搜索策略
2. **Inspect** 输出 `search_plan.md`
3. **Research** → **Search**: 根据 search_plan.md 执行查询
4. **Search** 输出原始材料到 `raw/` 文件夹

### 阶段 4: 分析与报告
1. **Research** → **Insight**: 根据原始材料编写报告
2. **Insight** 输出 `result.md`（调研报告）

### 阶段 5: 质量审核
1. **Research** → **Examine**: 根据 standard.md 审核 result.md
2. **Examine** 输出 `examine_report.md`（审核报告）

**审核结果判定**:
- **"大修"（Major Revision）**: 引用错误、重大事实错误、逻辑矛盾 → 进入阶段 6
- **"小修"（Minor Revision）**: 细节不足、格式问题 → 进入阶段 7
- **"通过"（Pass）**: 质量达标 → 进入阶段 8

### 阶段 6: 创造性扩展（大修时）
1. **Research** 将 examine 的改进要求写入 `revise.md`
2. **Research** → **Creative**: 根据 revise.md 提出新的调研方向
3. **Creative** 输出 `creative_directions.md`
4. **Research** 整合 creative_directions.md 和 request.md
5. **返回阶段 3**，进行扩展调研

### 阶段 7: 修订完善（小修时）
1. **Research** → **Insight**: 根据 examine 的反馈修改 result.md
2. **返回阶段 5**，重新提交审核
3. 循环直至 examine 判定"通过"

### 阶段 8: 定稿交付
1. **Research** 为报告添加元信息：
   - 当前日期
   - 调研周期次数
   - 总耗时
   - 所有查询来源清单
2. 输出最终报告 `final_report.md` 给用户

## 文档结构

调研过程中会生成以下文档：

```
workspace/
├── request.md              # 调研需求文档（Research 创建）
├── standard.md             # 审核标准文档（Research 创建）
├── search_plan.md          # 搜索计划（Inspect 创建）
├── raw/                    # 原始材料文件夹（Search 创建）
│   ├── 20250220_001_topic1.md
│   ├── 20250220_002_topic2.md
│   └── pdfs/               # 原始 PDF 文件
│       ├── 20250220_001_paper1.pdf
│       └── 20250220_002_paper2.pdf
├── result.md               # 调研报告（Insight 创建/修订）
├── examine_report.md       # 审核报告（Examine 创建）
├── revise.md               # 改进要求（Research 创建，基于 Examine 反馈）
├── creative_directions.md  # 新调研方向（Creative 创建）
└── final_report.md         # 最终报告（Research 创建）
```

## 使用方法

### 发起调研

向 **Research** 代理发起调研请求，提供以下信息：

1. **调研领域**: 例如"人工智能在医疗诊断中的应用"
2. **关键方向**: 具体关注的子领域或问题
3. **调研目标**: 希望回答的核心问题
4. **预期产出**: 报告形式、详细程度等（可选）

**示例**:
```
请帮我进行一项关于"中国新能源汽车市场发展"的深度调研。
关键方向：
- 市场规模和增长趋势
- 主要竞争者和市场份额
- 政策环境和技术发展
- 消费者偏好和购买行为

目标：全面了解中国新能源汽车市场的现状和未来发展趋势，为投资决策提供参考。
```

### 配合需求澄清

如果 Research 代理认为需求不够具体，会设计问卷与您确认。请尽可能详细地回答，这有助于提高调研质量。

### 接收最终报告

调研完成后，您将收到包含以下内容的 `final_report.md`：
- 执行摘要
- 调研目标和范围
- 详细发现和数据分析
- 结论和建议
- 所有引用来源清单
- 调研过程元信息

## 审核标准

### Examine 代理的审核维度

1. **引用完整性（Critical）**
   - 每个事实声明必须有引用
   - 引用内容必须与原始材料一致
   - **引用不符 → 必须"大修"**

2. **事实准确性**
   - 统计数据正确无误
   - 日期、名称等细节准确
   - 无虚假信息

3. **逻辑一致性**
   - 论证结构清晰
   - 结论基于证据
   - 无逻辑谬误

4. **完整性**
   - 回答所有调研问题
   - 无明显信息缺口
   - 承认不确定性和矛盾

5. **标准合规性**
   - 符合 standard.md 的要求
   - 专业写作标准
   - 格式规范

### 三级修订制度

- **大修（Major Revision）**: 存在引用错误、重大事实错误或逻辑问题，需要补充调研
- **小修（Minor Revision）**: 内容基本正确，但需要完善细节、优化表达或调整结构
- **通过（Pass）**: 完全符合标准，可直接定稿

## 最佳实践

1. **需求尽量具体**: 清晰的需求能显著提高调研效率和质量
2. **及时响应问卷**: 在需求澄清阶段积极配合
3. **保留原始材料**: Search 代理保存的 raw/ 文件夹包含所有原始信息，可供核实
4. **利用修订机制**: 如对报告不满意，可以通过反馈触发修订流程
5. **关注引用**: 所有结论都有来源支持，可追溯验证

## 注意事项

1. **调研周期**: 复杂调研可能需要多个周期（需求澄清→查询→分析→审核→修订）
2. **信息时效性**: Search 代理获取的信息截止于查询日期，请注意时效性
3. **PDF 文件**: 学术论文等重要资料会保存原始 PDF，确保信息完整性
4. **Creative 代理**: 温度设置较高（0.7），可能提出非常规但富有价值的调研方向
5. **Examine 代理**: 温度设置极低（0.0），审核标准极其严格，仅相信数学和逻辑

## 故障排查

### 调研时间过长
- 需求可能过于宽泛，建议缩小范围
- 检查是否需要更多轮次的修订

### 报告质量不理想
- 检查 standard.md 是否定义了明确的标准
- 考虑是否需要 creative 代理扩展调研方向
- 提供更具体的反馈给 Research 代理

### 引用问题
- Examine 代理会严格检查每个引用
- 确保 Search 代理保存的 raw/ 材料完整
- 如发现问题，会要求"大修"并补充调研

## 技术细节

### 代理配置

所有代理定义文件位于 `agents/` 目录：
- `research.md` - 主代理配置
- `inspect.md` - 查询设计代理配置
- `search.md` - 信息收集代理配置
- `insight.md` - 报告编写代理配置
- `creative.md` - 创造性思考代理配置
- `examine.md` - 质量审核代理配置

### 温度设置说明

- **0.0（Examine）**: 完全客观，无创造性，严格基于事实
- **0.3（Inspect/Search）**: 较低温度，精确执行，减少随机性
- **0.4（Insight）**: 中等温度，分析性思考，平衡客观与综合
- **0.5（Research）**: 中等温度，协调管理，灵活应变
- **0.7（Creative）**: 较高温度，鼓励创新和发散思维
